2 ЗАВДАННЯ
     2.1 Мета роботи:
Навчитися розгортати програмну систему, перевіряти її функціональність, виконувати налаштування, а також демонструвати та документувати роботу системи.
   2.2 Завдання:
1. Розгорнути створену програмну систему:
o Використовуючи інструменти для розгортання (наприклад, Docker, Kubernetes, хмарні сервіси), розгорнути серверну частину, веб-клієнт, мобільний клієнт та IoT клієнт.
o Налаштувати середовище для роботи програмної системи (сервер, база даних, мережеві налаштування).
2. Перевірити та налаштувати роботу розгорнутої програмної системи:
o Провести перевірку стабільності та коректності роботи усіх компонентів програмної системи (серверна частина, IoT клієнт, база даних, API).
o Виконати налаштування параметрів серверної частини, клієнтських застосунків та мережевих протоколів для коректної роботи системи.
3. Продемонструвати описану у розділі 2.2 Vision & Scope функціональність програмної системи:
o Описати та продемонструвати всі функції, передбачені в розділі 2.2 Vision & Scope, включаючи бізнес логіку, функції адміністрування та взаємодію між серверною частиною та клієнтами.
o Перевірити реалізацію всіх необхідних функціональних можливостей програмної системи.




3 ОПИС ВИКОНАНОЇ РОБОТИ
    Для початку роботи з хмарним середовищем Azure було створено безкоштовний акаунт з відповідним Billing Plan, що дозволяє тестувати та розгортати ресурси без фінансових вкладень.
    Далі була створена ресурсна група, яка слугує контейнером для всіх пов’язаних ресурсів проекту.
    Для хостингу веб-додатку було розгорнуто Azure Web App, що забезпечує готову платформу для публікації .NET додатків без потреби налаштовувати сервери вручну. Перевагою цього підходу є автоматичне масштабування, управління сертифікатами SSL та інтеграція з DevOps інструментами.
    База даних була реалізована за допомогою Azure SQL Server та SQL Database, що дозволяє використовувати потужності реляційної СУБД у хмарі. Було налаштовано доступи та дозволи для безпечного підключення додатку до бази даних.
    Публікація додатку здійснювалася безпосередньо з середовища розробки використовуючи вбудовані інструменти для розгортання на Azure Web App. Це дозволило швидко перевести проект з локального середовища на хмарний сервер без складних налаштувань.
  Серед основних переваг Azure, які були використані у цьому процесі: простота інтеграції з Visual Studio та Raider, наявність безкоштовного тарифу для тестування, автоматичне масштабування додатків, високий рівень безпеки та резервного копіювання, а також можливість централізованого управління всіма ресурсами через портал Azure або командний рядок.


4 ВИСНОВКИ
     Під час виконання роботи було здобуто практичні навички розгортання програмної системи на різних платформах та перевірки її функціональності. Було успішно налаштовано серверну частину, веб- та мобільний клієнти, а також IoT-клієнт, забезпечено коректну роботу бази даних та налаштувань мережевих протоколів, що дозволило системі стабільно виконувати всі передбачені функції.
     Крім того, було перевірено реалізацію бізнес-логіки, функцій адміністрування та взаємодії між сервером та клієнтами.
